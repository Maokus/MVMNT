# State Baseline Decisions (Phase 0)

Date: 2025-09-17

Purpose: Capture initial assumptions and defaults to prevent drift during refactor.

IDs

-   Track IDs: `trk_<random>` generated by `makeId()` in `timelineStore`.
-   Timeline ID default: `tl_1`.
-   Scene metadata id during export defaults to `timeline.id` or `scene_1` fallback.

Time Units

-   Canonical playhead unit: ticks.
-   PPQ: defined by shared `TimingManager` (`CANONICAL_PPQ`), currently inferred from imports.
-   Tempo conversion: `beats <-> seconds` via `tempo-utils` with fallback `globalBpm` when no tempo map.

UI Defaults

-   `rowHeight`: 30.
-   `timelineView`: start `0`, end `~120 beats` in ticks.
-   Transport: `isPlaying=false`, `loopEnabled=false`, `rate=1.0`, `quantize='bar'`.
-   Loop range initial: derived from seconds helper (~2s to ~5s converted to beats -> ticks).

Persistence Envelope v1

-   `schemaVersion=1`, `format='mvmnt.scene'`.
-   Includes: `metadata`, `scene` (elements, optional sceneSettings/macros), `timeline` (timeline, tracks, tracksOrder, transport, selection, timelineView, playbackRange\*, rowHeight, midiCache).
-   Compatibility section with `warnings` array.

Notes

-   Import currently hydrates both UI and document-like fields via `useTimelineStore.setState`.
-   Undo: Legacy snapshot-based controller has been removed in Phase 6; use documentStore patch-based undo via actions.

Validation

-   `validateSceneEnvelope` performs structural checks (Phase 2 error codes already present).

Next

-   Future phases will hide document state behind a gateway and patch-based undo.
